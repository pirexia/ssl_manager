## Login & MFA

### Logging In
1.  Navigate to the application URL (e.g., `http://your-server:8000`).
2.  Enter your **Username** and **Password**.
3.  Select the **Authentication Source** (Local or LDAP) if configured.
4.  Click **Sign In**.

### Multi-Factor Authentication (MFA)
Upon first login, you may be asked to set up MFA:
1.  Scan the QR code with an authenticator app (Google Authenticator, Authy, etc.).
2.  Enter the 6-digit token generated by the app.
3.  **Trust this device**: Check this box to skip MFA for 7 days on this browser.

## Dashboard
The **Home** page serves as your dashboard. It provides quick access to:
- **Generate New**: Create a new Certificate Signing Request (CSR).
- **Search Repository**: Find existing certificates.
- **Recent Activity**: (If implemented) View recently created certificates.

---

## Generating a CSR
To request a new SSL certificate:
1.  Click **Generate New** on the navigation bar or home page.
2.  **Domain**: Select the base domain (e.g., `example.com`) from the dropdown.
3.  **Subdomain**: Enter the subdomain (e.g., `www`, `api`).
4.  **Key Size**: Default is 2048 bits (recommended).
5.  **Country**: Select your country code (e.g., ES, US).
6.  Click **Generate CSR**.

**Result:**
- You will see the generated **CSR** (text block starting with `-----BEGIN CERTIFICATE REQUEST-----`).
- You will see the **Private Key** (keep this secure!).
- You can **Download** these files immediately.

## Searching Certificates
To find a certificate:
1.  Click **Search Repository**.
2.  **Search Bar**: Type a common name, domain, or serial number. The results update automatically.
3.  **Filters**:
    - **Domain**: Filter by specific domains.
    - **Status**: Filter by Valid, Expired, or Pending.
4.  Click **View** on any result to see details.

---

## Managing Certificates
On the **Certificate Detail** page:
- **Overview**: View validity dates, issuer, and subject.
- **Download**: Download the certificate in various formats (.crt, .cer, .pem, .pfx).
- **Upload Certificate**: If you have the signed certificate from a public CA (like DigiCert or Sectigo), upload it here to complete the entry.
- **History**: View previous versions of certificates for this common name.

---

## Internal CA
For internal use (dev/test environments), you can sign certificates yourself:
1.  Go to the **Certificate Detail** page of a pending CSR.
2.  Click **Generate Internal Certificate**.
3.  The system will sign the CSR using its internal Root CA.
4.  The status will change to **Issued**.
5.  You can now download the certificate and the **CA Bundle** (Root + Intermediate) to trust it in your browser/OS.
